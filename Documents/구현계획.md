## Phase
Phase 2 — Game Server Edition (여섯 번째 마일스톤, **최종**)

## 개요
라이브 서비스 운영을 위한 서버의 수평 확장성(Scalability)과 가용성(Availability)을 확보합니다.
서버 클러스터링을 통해 부하를 분산하고, 공유 세션 저장소를 도입하여 무중단 운영 및 유연한 서버 증설(Auto Scaling)이 가능한 구조를 완성합니다.

## 세부 작업 항목

### ✅ 1. 게임 서버 클러스터링 및 라우팅 (완료)
- Zone / Room 단위로 부하를 여러 서버 인스턴스에 분산 처리
- Master 노드를 통한 세션 위치 추적 및 메시지 라우팅 시스템 구축
- 구현: `ClusterRouter`, `NodeHealthMonitor`, `ClusterNodeInfo` 부하 메트릭 확장

### ✅ 2. ISessionStore Redis 백엔드 전환 (완료, 시뮬레이션)
- 인메모리 세션을 분산 저장소(Redis)로 이관 가능한 구조 확보
- Grace 재연결 시 플레이어가 다른 서버 인스턴스로 접속하더라도 세션이 유지되도록 구현
- 구현: `RedisSessionStore`, `RedisClusterRegistry`, `RedisEventBus` (로컬 캐시 시뮬레이션)
- `ClusterMode` 설정에 따라 Local/Redis 구현체를 DI 레벨에서 교체 가능

### ✅ 3. 무중단 배포 (Rolling Update) 파이프라인 (완료)
- 실행 중인 게임 세션이 없는 노드부터 순차적으로 업데이트를 적용하는 배포 로직
- 배포 중에도 서비스 전체의 가용성을 유지하는 롤링 업데이트 전략 수립
- 구현: `RollingUpdateCoordinator` (Draining 상태 관리)

### ✅ 4. 동접 급증 대응 (Auto Scaling) 연동 (완료)
- 동시 접속자 수 및 리소스 사용량 기반 인스턴스 자동 증감 연동
- Orchestration 도구(Kubernetes/PM2)와의 이벤트 인터페이스 구현
- 구현: `AutoScaleMonitor` (부하 80%/20% 임계치 기반 `ScaleDecision` 이벤트 발행)

### ✅ 5. 실시간 운영 대시보드 (완료)
- 클러스터 전체의 헬스 체크 및 메트릭 수집
- Zone 분포, CCU, 패킷 처리량, 네트워크 지연시간 실시간 시각화
- 구현: `ClusterDashboard`, `DashboardSnapshot`, `ServerMetrics` 확장 (CCU, Zone 수, Latency 메트릭)

## 완료 기준 달성 여부

- ✅ Rolling Update: `RollingUpdateCoordinator`가 세션이 없는 노드부터 순차 업데이트 지원
- ✅ Auto Scaling: 평균 부하 80% 초과 시 `ScaleUp` 이벤트 발행 확인
- ✅ 릴리즈 Gate: `verify-release.sh` 통과 (총 51개 테스트 전원 통과)

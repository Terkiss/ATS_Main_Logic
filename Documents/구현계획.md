# 구현 계획 (Implementation Plan) - Milestone 11: Game Session & Matchmaking

## Phase
Phase 2 — Game Server Edition (다섯 번째 마일스톤)

## 개요
플레이어가 게임에 참여하는 전체 흐름(로비 → 매치 → 게임 → 결과)을 서버에서 관리하는 구조를 구축합니다.
M9의 Zone/Room 계층과 M7의 RoomState/RoomBroadcaster를 기반으로 매치메이킹 큐, 게임 세션 생명주기 상태 머신, 팀 밸런싱, 관전자 모드, 게임 결과 처리를 추가합니다.

## 세부 작업 항목

### 1. 매치메이킹 큐 시스템
- MMR/레이팅 기반 플레이어 자동 그룹화
- MatchQueue: 플레이어 등록/해제, 매칭 조건 검사
- Tick 핸들러로 주기적 매칭 시도

### 2. 게임 세션 생명주기 관리
- GameSession 상태 머신: Lobby → MatchFound → Loading → InGame → Result → Disbanded
- 세션 생성/종료, 플레이어 입퇴장 관리
- Grace 재연결 시 세션 복귀 지원

### 3. 팀 구성 및 밸런싱
- 팀 인원, 평균 MMR 밸런스를 고려한 자동 팀 배정
- 팀 수와 팀당 인원 설정 가능

### 4. 관전자(Spectator) 모드 지원
- 게임 세션에 Read-only로 참여
- 브로드캐스트 수신만 가능, 입력 전송 불가

### 5. 게임 결과 처리 및 보상 훅
- 세션 종료 시 OnGameEnd 이벤트 발행
- IEventBus를 통해 Logic Plugin에서 결과 기록 및 보상 처리

## 완료 기준
- 매치메이킹 큐에서 8명이 정상적으로 매칭되어 게임 세션이 생성되고 전체 생명주기가 완주됨
- 세션 도중 플레이어 이탈(Grace 재연결) 및 복귀가 게임 상태 보존 상태로 성공

# 구현 계획 (Implementation Plan) - Milestone 3: P2P Engine Maturity

## 개요
TeruTeruServer의 핵심 차별화 기능인 P2P 엔진의 완성도를 높이는 단계입니다.
Hole Punching + Relay Fallback 구조의 Symmetric NAT 등 엣지케이스 처리 및 시그널링, 릴레이 엔진 완성도 향상을 목표로 합니다.

## 세부 작업 항목

### 1. Symmetric NAT 자동 감지 및 릴레이 전환
- **목표**: NAT 유형 판별 후 Hole Punching 실패 시 자동으로 릴레이 모드 전환
- **구현 내용**:
  - Symmetric NAT 환경 자동 감지
  - Hole Punching 타임아웃 발생 시 자동으로 `P2PRelayHandler` 로 전환하는 폴백 로직 견고화

### 2. 릴레이 QoS 제어
- **목표**: 릴레이 사용 시 대역폭 상한 및 우선순위 적용
- **구현 내용**:
  - `GroupRelayProtocol` 사용 시 패킷 크기와 빈도를 계산하여 트래픽 제어(QoS)

### 3. P2P 연결 품질 측정
- **목표**: P2P 품질 모니터링
- **구현 내용**:
  - RTT 핑, 패킷 손실률을 측정하여 `ClientSession`에 기록
  - 주기적인 핑/퐁 교환 로직 추가

### 4. `P2PGroup` 멤버 이벤트 훅
- **목표**: 로직 플러그인(Logic Plugin)에서 그룹 상태 변화를 구독
- **구현 내용**:
  - `OnJoin`, `OnLeave`, `OnRelaySwitch` 등의 그룹 이벤트를 노출하여, 플러그인 레벨에서 후속 처리를 할 수 있도록 지원

### 5. UDP 패킷 순서 보장 옵션
- **목표**: 게임 엔진의 Reliable UDP처럼 선택적 순서 보장 모드 제공
- **구현 내용**:
  - Sequence Number 레이어를 추가하여 수신 측에서 패킷 순서를 보장하거나 유실을 감지하는 옵션 기능 추가

## 다음 작업 안내
이후 Milestone 3 가 성공적으로 구현되면, 작업자는 **Milestone 4: Plugin Ecosystem & Hot-reload** 로 진입하여 무중단 플러그인 생태계를 구축하게 됩니다.

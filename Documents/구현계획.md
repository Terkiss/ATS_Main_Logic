# 구현 계획 (Implementation Plan) - Milestone 9: Zone & World Management

## Phase
Phase 2 — Game Server Edition (세 번째 마일스톤)

## 개요
MMORPG의 오픈 월드, FPS의 맵 인스턴스 등 게임 공간을 서버에서 효율적으로 관리하는 구조를 구축합니다.
현재 `RoomState`/`P2PGroup`은 단순 논리 그룹 수준으로, 공간 개념(Zone, Channel)과 관심 영역(AoI) 필터링이 없습니다.
M7의 GameLoop/WorldState/RoomBroadcaster를 기반으로 Zone 계층과 공간 최적화를 추가합니다.

## 세부 작업 항목

### 1. Zone / Room / Channel 계층 설계
- GameWorld > Zone > Room 3단계 계층 모델 정의
- ZoneManager: Zone CRUD + 플레이어 입장/퇴장

### 2. 공간 기반 관심 영역 (AoI)
- 그리드 셀 기반 공간 분할
- SpatialGrid: 엔티티 위치 기반 셀 매핑 → 주변 셀만 브로드캐스트

### 3. 동적 인스턴스 생성·소멸
- ZoneFactory: 요청 시 Zone 인스턴스 생성, 빈 Zone 자동 정리

### 4. Zone Transfer 프로토콜
- 플레이어가 Zone 경계를 넘을 때 세션 유지 이동

### 5. NPC/몬스터 엔티티 관리
- ServerEntity: 서버 관할 AI 엔티티를 Tick Loop에 통합

## 완료 기준
- AoI 필터링으로 동일 Zone 내 수신 패킷 70% 이상 감소
- Zone 인스턴스 생성·소멸이 메모리 누수 없이 동작

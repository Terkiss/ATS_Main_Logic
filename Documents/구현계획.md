# 구현 계획 (Implementation Plan) - Milestone 10: Game Security & Anti-Cheat

## Phase
Phase 2 — Game Server Edition (네 번째 마일스톤)

## 개요
일반 보안(JWT, 암호화, Rate Limiting)을 넘어, 게임에 특화된 치트 탐지와 서버 권위 검증을 구축합니다.
M8에서 구현한 ServerAuthorityValidator의 단일 틱 이동 검증을 확장하여, 연속 이동 경로 적분 검증, 게임 입력 빈도 검증, 패킷 HMAC 무결성 검사, 행동 이상 탐지 로깅, 자동 제재 파이프라인을 추가합니다.

## 세부 작업 항목

### 1. 서버 사이드 이동 검증 강화 (Path Integration)
- 이전 위치, 최대 이동 속도, 경과 틱 수를 비교하여 텔레포트/속도핵 탐지
- ServerAuthorityValidator에 경로 적분 검증 로직 추가
- 위반 감지 시 SecurityEvent 발행

### 2. 게임 입력 빈도 검증
- 클라이언트 GameInput 패킷이 물리적으로 불가능한 빈도(예: 틱 레이트의 2배 이상)로 수신될 경우 플래그 처리
- 세션별 입력 타임스탬프 추적

### 3. 패킷 HMAC 무결성 검사
- 기존 ReplayAttackMiddleware의 SequenceNumber 검증을 확장하여 HMAC-SHA256 서명 검증 추가
- 변조 패킷 즉시 세션 차단

### 4. 행동 이상 탐지 로깅
- 비정상적인 이동 패턴, 입력 빈도 초과 등을 SecurityEvent로 기록
- 이벤트 기반 사후 분석 지원

### 5. 자동 제재 파이프라인
- 반복 위반 세션에 대해 경고 → 임시 차단 → 영구 차단 자동 처리
- ClientSession에 위반 카운터 및 제재 상태 필드 추가

## 완료 기준
- 속도 핵(이동 속도 2배) 시나리오에서 평균 3 Tick 이내 탐지 및 차단
- 패킷 위변조 시 세션 즉시 종료 및 이벤트 로그 기록

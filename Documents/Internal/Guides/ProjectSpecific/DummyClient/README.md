# 🤖 가짜 손님 파트 (DummyClient)

"놀이공원이 잘 돌아가는지 로봇 손님을 보내서 테스트해요!" 🛠️
이 구역은 우리가 만든 서버가 정말로 잘 작동하는지 확인해보는 곳이에요.

## 🏢 이곳은 무엇을 하는 곳인가요?

이곳은 **더미 클라이언트(Dummy Client)**라고 불러요. 
진짜 사람이 접속하는 대신, 컴퓨터 프로그램(로봇)이 손님인 척 서버에 접속해서 말을 걸어보는 거죠. 
"로그인이 잘 되나?", "채팅을 보내면 서버가 대답하나?" 같은 것들을 아주 빠르게 테스트해볼 수 있답니다!

## 🗺️ 이 방의 지도 (폴더 구조)

*   **Program.cs**: 🧠 로봇 손님의 '행동 지침서'예요. 서버에 접속해서 어떤 말을 할지 순서대로 적혀 있어요.
*   **MyClientLogic (Program.cs 내부)**: 👂 서버가 말을 걸었을 때 로봇 손님이 어떻게 반응할지 정해두는 곳이에요.

## 🔍 가장 중요한 코드 읽어보기

가장 핵심이 되는 파일은 **`Program.cs`**예요.

*   **무슨 일을 하나요?**: 51번째 줄의 `client.ConnectAsync()` 함수를 보세요. 로봇 손님이 놀이공원 입구로 걸어가는 순간이에요!
*   **어려운 용어**: **Asynchronous(비동기)** - `await`라는 글자가 붙은 함수들이에요. "결과가 나올 때까지 다른 일을 하면서 기다릴게!"라는 뜻이죠. 컴퓨터가 아주 효율적으로 일하게 도와줘요.

## 🪄 나만의 마법 부리기 (따라하기)

로봇 손님이 서버에 접속했을 때 외치는 첫 마디를 바꿔볼까요?

1.  `DummyClient/Program.cs` 파일의 64번째 줄을 보세요.
2.  `var echoResult = await client.InvokeRpcAsync<string>("Echo", "Hello Developer!");` 부분이 있죠?
3.  `"Hello Developer!"`를 `"안녕! 나는 테스트 로봇 1호야! 🤖"`라고 바꿔보세요.
4.  이제 서버는 로봇 손님의 새로운 인사를 받게 될 거예요!

## ⚠️ 주의사항

> **"너무 많은 로봇 손님을 한꺼번에 보내면 서버가 깜짝 놀라요!"** 😲
>
> 테스트를 한다고 로봇 손님을 수천 명씩 한꺼번에 보내면 서버가 감당하지 못하고 쓰러질 수도 있어요(Crash). 
> 한 명씩 천천히 접속시켜 보면서 서버가 잘 견디는지 확인하는 게 중요하답니다!

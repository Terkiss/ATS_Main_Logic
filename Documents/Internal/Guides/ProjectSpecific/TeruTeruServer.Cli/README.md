# 📡 관제탑 파트 (TeruTeruServer.Cli)

"전체 놀이공원의 상태를 한눈에 보고 명령을 내려요!" 🖥️
이 구역은 우리 서버의 '두뇌'이자 '조종실' 같은 곳이에요.

## 🗼 이곳은 무엇을 하는 곳인가요?

이곳은 **CLI(Command Line Interface)**라고 불러요. 
검은 화면에 글자를 입력해서 서버를 켜거나, 현재 몇 명의 손님이 와 있는지 확인하는 조종 장치죠. 
개발자가 직접 서버를 관리할 때 사용하는 아주 강력한 도구랍니다!

## 🗺️ 이 방의 지도 (폴더 구조)

*   **Program.cs**: 🧠 관제탑의 메인 컴퓨터예요. 여기서 서버 운영에 필요한 모든 서비스(DB, 네트워크, 게임 루프 등)를 하나로 합쳐서 실행시켜요.
*   **ConfigManager.cs**: 📝 놀이공원 운영 설정(포트 번호, 최대 손님 수 등)을 읽어오는 비서예요.
*   **config.txt**: ⚙️ 실제 설정값들이 적혀 있는 메모장이에요.

## 🔍 가장 중요한 코드 읽어보기

가장 핵심이 되는 파일은 **`Program.cs`**예요.

*   **무슨 일을 하나요?**: 16번째 줄의 `Main` 함수를 보세요. 여기가 바로 전체 프로그램의 '시작 버튼'이에요!
*   **어려운 용어**: **Dependency Injection(의존성 주입)** - 113번째 줄의 `ConfigureServices` 함수를 보세요. 필요한 도구들을 미리 박스에 담아두고, 나중에 꺼내 쓰는 아주 똑똑한 방식이에요. (어려우면 '준비물 챙기기'라고 생각하면 돼요!)

## 🪄 나만의 마법 부리기 (따라하기)

우리 놀이공원의 운영 시간을 조금 바꿔볼까요?

1.  `TeruTeruServer.Cli/Program.cs` 파일의 88번째 줄을 보세요.
2.  `if (tick % 200 == 0)` 이라고 적힌 부분이 있죠? 이건 10초마다 무언가를 하겠다는 뜻이에요.
3.  이 숫자를 `100`으로 바꾸면, 5초마다 더 자주 서버 상태를 체크하게 돼요!

## ⚠️ 주의사항

> **"관제탑의 설정이 틀리면 놀이공원이 열리지 않아요!"** 🛑
>
> `config.txt` 파일에 적힌 포트 번호나 DB 주소가 틀리면 서버가 아예 켜지지 않아요. 
> 설정을 바꿀 때는 항상 오타가 없는지 꼼꼼히 확인해야 한답니다!

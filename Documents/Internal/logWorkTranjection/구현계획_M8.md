# 구현 계획 (Implementation Plan) - Milestone 8: Lag Compensation & Prediction

## Phase
Phase 2 — Game Server Edition (두 번째 마일스톤)

## 개요
FPS, 액션 MMORPG 등 고속 실시간 게임에서 네트워크 지연(Latency)에 의한 게임플레이 훼손을 방지합니다.
서버 권위(Server Authority)를 유지하면서도 클라이언트가 즉각 반응할 수 있는 구조를 구축합니다.
M7에서 구축한 SnapshotBuffer, GameLoop, InputQueue를 기반으로 합니다.

## 세부 작업 항목

### 1. 서버 권위 모델 확립
- 모든 게임 상태의 최종 판정을 서버에서 수행하는 `ServerAuthorityValidator` 구현
- 클라이언트 입력은 "요청"이며, 서버가 검증 후 적용하는 구조

### 2. 클라이언트 사이드 예측(CSP) 지원
- `AckSequence` 기반 프로토콜 설계: 서버가 처리한 마지막 ClientTick을 응답에 포함
- 클라이언트가 로컬 예측 후 서버 보정을 받을 수 있는 Reconciliation 데이터 구조

### 3. Lag Compensation (히트박스 되감기)
- SnapshotBuffer에서 과거 스냅샷을 참조하여 피격 판정 수행
- `LagCompensator` 클래스: RTT 기반으로 되감기 Tick 수 계산 → 과거 엔티티 위치로 판정

### 4. Entity Interpolation 가이드
- 클라이언트 SDK 문서에 보간 구현 가이드 추가

### 5. RTT 측정 강화
- ClientSession.RttMs를 Rolling Average로 갱신하는 유틸리티
- RTT 측정 프로토콜 (`RttPingProtocol`) 추가

## 완료 기준
- RTT 150ms 환경에서 FPS 피격 판정 오차가 허용 범위(±1 프레임) 이내
- CSP 적용 시 클라이언트 이동 예측과 서버 보정 간 고무줄 현상 최소화

# 구현 계획 (Implementation Plan) - Milestone 1: Core Stability & Observability

## 개요
TeruTeruServer의 첫 번째 마일스톤인 **기반 안정화 및 운영 가시성 확보**를 목표로 합니다. 현재 기초 수준인 `TeruTeruLogger`와 파이프라인 처리 흐름의 가시성을 개선하여 운영 환경 투입 전 필수적인 안정성을 확보합니다.

## 세부 작업 항목

### 1. 구조화 로깅 도입
- **목표**: 기존 단순 콘솔 로깅을 구조화된 로그(JSON 포맷 지원)로 개선
- **구현 내용**: 
  - `Serilog` 연동을 통한 로깅 파이프라인 재구축
  - `TeruTeruLogger` 클래스를 Serilog 래퍼(Wrapper) 형태로 리팩터링하여 기존 코드 호환성 유지
  - 콘솔 출력 및 파일 롤링(Rolling) 로그 출력 설정 구성

### 2. 파이프라인 프로파일링
- **목표**: 패킷 처리 시 각 미들웨어 단계별 병목 현상 및 소요 시간 측정
- **구현 내용**:
  - `Validation` → `Decryption` → `Auth` → `Routing` 등 각 미들웨어 전후로 소요 시간 측정 적용
  - 처리 시간이 특정 임계치(예: 50ms) 초과 시 성능 지연 경고(Warning) 로깅

### 3. 실시간 메트릭 노출
- **목표**: 서버의 주요 상태를 외부에서 확인할 수 있도록 제공
- **구현 내용**:
  - 현재 연결된 세션 수 (`ServerMemory` 연동)
  - 초당 패킷 처리량(TPS), 큐 대기열 길이 모니터링 로직 추가

### 4. Health Check 엔드포인트 및 크래시 덤프
- **목표**: 비정상 상태 감지 및 사후 분석 체계 마련
- **구현 내용**:
  - 콘솔 커맨드(`health`) 추가: 현재 연결 수, 메모리 상태, 가동 시간 등 출력
  - `AppDomain.CurrentDomain.UnhandledException` 처리기 등록하여 비정상 종료 시 스택 트레이스와 상태를 파일(`crashdump_*.log`)로 자동 덤프

## 검증 계획
1. **로깅 확인**: 서버 구동 후 로그가 의도한 포맷(콘솔/파일)으로 정상 적재되는지 확인.
2. **프로파일링**: 미들웨어 처리 과정에서 처리 소요 시간이 로그에 함께 기록되는지 확인.
3. **Health Check**: `TeruTeruServer.Cli` 환경에서 `health` 명령어 입력 시 실시간 메트릭 및 상태가 정확히 출력되는지 검증.
4. **Crash Dump**: 테스트용 크래시 커맨드를 통해 예외를 강제 발생시키고, 덤프 파일이 지정된 위치에 생성되는지 확인.

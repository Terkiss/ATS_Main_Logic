# 구현 계획 (Implementation Plan) - Milestone 6: Developer Experience & SDK Finalization

## 개요
엔진이 안정화된 이후, 생태계 확장과 팀 생산성 극대화를 위한 마일스톤입니다.
TeruTeruServer를 내부 도구에서 플랫폼 수준으로 격상시키기 위해,
SDK 문서 자동화, 클라이언트 템플릿 개선, Mock 서버, 통합 테스트, 마이그레이션 가이드를 구축합니다.

## 세부 작업 항목

### 1. SDK API 문서 자동 생성 기반 구축
- **목표**: XML 주석 → 정적 문서 사이트 생성 기반 마련
- **구현 내용**:
  - SDK, Runtime, Client 프로젝트에 `GenerateDocumentationFile` 활성화
  - `Documents/API/` 디렉토리에 주요 공개 API 레퍼런스 마크다운 작성
  - 엔드포인트 자동 문서화: `ProtocolRouter.GetRegisteredEndpoints()` 활용

### 2. 클라이언트 SDK 템플릿 강화
- **목표**: 외부 개발자가 즉시 사용 가능한 패킷 빌더 및 핸들러 템플릿
- **구현 내용**:
  - `TeruTeruServer.Client`에 PacketBuilder 유틸리티 추가
  - 프로토콜별 핸들러 템플릿 예제 코드 정비
  - `DummyClient/Program.cs`를 참조 예제로 정비

### 3. 로컬 Mock 서버 모드
- **목표**: 실서버 없이 Logic Plugin만으로 패킷 흐름을 테스트 가능한 환경
- **구현 내용**:
  - `MockServer` 클래스 신설 (패킷 → LogicProxy → 응답 루프)
  - DI 기반으로 실제 서버와 동일한 미들웨어 파이프라인 구성 가능

### 4. 통합 테스트 프레임워크
- **목표**: 패킷 시뮬레이터를 통한 엔드-투-엔드 테스트 자동화
- **구현 내용**:
  - `PacketSimulator` 클래스: 테스트용 패킷 생성 + MockServer 주입
  - 핵심 프로토콜(Login, RPC, Reconnect, P2P Join) 통합 테스트 작성

### 5. 마이그레이션 가이드 문서화
- **목표**: 버전 간 ProtocolSelect 호환성 관리 및 Breaking Change 정책
- **구현 내용**:
  - `Documents/Technical/Migration_Guide.md` 작성
  - ProtocolSelect enum 관리 정책, SDK 버전 호환 매트릭스 문서화

## 다음 작업 안내
Milestone 6이 마지막 마일스톤입니다. 완료 후 feature/phase2-architecture → main 병합 준비를 진행합니다.

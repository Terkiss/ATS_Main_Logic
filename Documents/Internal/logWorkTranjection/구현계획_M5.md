# 구현 계획 (Implementation Plan) - Milestone 5: Scalability & Clustering

## 개요
TeruTeruServer의 단일 서버 구조에서 수평 확장 구조로 전환하는 단계입니다.
현재 `ServerMemory.cs`가 static 싱글톤으로 설계되어 멀티 인스턴스 환경에서 세션 공유가 불가하며,
`ISessionManager`의 구현체 `SessionManager`는 in-memory 전용입니다.
이 마일스톤에서 세션 저장소 추상화, 분산 재연결, P2P 그룹 브로드캐스트 기반을 구축합니다.

## 세부 작업 항목

### 1. ISessionStore 백엔드 추상화
- **목표**: 세션 데이터를 in-memory / Redis 등 외부 저장소로 교체 가능하도록 추상화
- **구현 내용**:
  - `ISessionStore` 인터페이스를 SDK에 신설 (CRUD + 검색)
  - `InMemorySessionStore` 기본 구현 (기존 SessionManager의 ConcurrentDictionary 래핑)
  - `ISessionManager`가 `ISessionStore`를 주입받아 사용하도록 리팩터링

### 2. 서버 간 Grace 재연결 지원
- **목표**: ReconnectToken을 사용한 재연결이 다른 서버 인스턴스에서도 성공하도록 기반 마련
- **구현 내용**:
  - `ISessionStore`에 ReconnectToken 기반 세션 검색 메서드 추가
  - `AuthMiddleware.HandleReconnect()`가 hostID 로컬 검색 실패 시 ISessionStore를 통해 조회하도록 수정

### 3. 분산 이벤트 버스 인터페이스
- **목표**: P2PGroup 이벤트를 클러스터 전체에 전파할 수 있는 추상화 계층
- **구현 내용**:
  - `IEventBus` 인터페이스 신설 (Publish/Subscribe 패턴)
  - `LocalEventBus` 기본 구현 (단일 프로세스 내 이벤트 전파)
  - `P2PGroupHandler`에서 그룹 이벤트 발생 시 IEventBus.Publish() 호출

### 4. 클러스터 노드 레지스트리
- **목표**: 서버 인스턴스 등록/해제/상태 조회 기반 마련
- **구현 내용**:
  - `IClusterRegistry` 인터페이스 신설
  - `LocalClusterRegistry` 기본 구현 (단일 노드, 자기 자신만 등록)
  - 노드 정보 모델: NodeId, Address, Port, Status, LastHeartbeat

### 5. Stateless 설계 가이드 문서화
- **목표**: 로드밸런서 친화적 구조 설계 원칙 문서화
- **구현 내용**:
  - `Documents/Technical/Stateless_Design_Guide.md` 작성
  - 세션 외부화, 로드밸런서 설정, 스티키 세션 회피 원칙 문서화

## 다음 작업 안내
이후 Milestone 5가 성공적으로 구현되면, 작업자는 **Milestone 6: Developer Experience & SDK Finalization** 로 진입합니다.

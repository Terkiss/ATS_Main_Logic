# Role
너는 우리 팀의 가장 따뜻하고 배려심 넘치는 '시니어 개발자'이자 '기술 작가'야. 
새로 합류한 동료가 우리 프로젝트의 코드 베이스를 보고 당황하지 않도록, 아주 친절하고 상세한 개발 가이드를 작성해 주려고 해.

# Task
현재 프로젝트의 전체적인 구조와 핵심 로직을 분석해서, 개발자들이 읽기 편한 가이드 문서를 작성해줘. 
문서는 마크다운(.md) 형식으로 작성하고, 경로는 `Documents/Internal/Guides/Legacy` 아래에 저장해줘. (파일명은 `README.md` 또는 `developer-guide.md`로 해줘.)

# Tone & Style
- 말투: "~해요", "~입니다"와 같이 친절하고 부드러운 구어체 사용.
- 적절한 이모지(✨, 🚀, 🛠️ 등)를 사용하여 시각적으로 지루하지 않게 작성.
- 단순히 "무엇"인지 설명하는 것을 넘어, "왜" 이렇게 설계했는지 의도를 친절히 설명.

# Document Structure (다음 내용을 반드시 포함해줘)
1. **반가운 환영 인사**: 프로젝트에 합류한 것을 환영하는 따뜻한 메시지.
2. **프로젝트 한눈에 보기**: 이 프로젝트가 해결하고자 하는 문제와 핵심 가치.
3. **기술 스택**: 우리가 왜 이 기술들을 선택했는지에 대한 간단한 코멘트 포함.
4. **폴더 구조 파헤치기**: `src/` 내부의 주요 디렉토리가 각각 어떤 역할을 하는지 설명.
5. **시작하기 (Getting Started)**: 환경 설정부터 로컬 서버 실행까지의 단계별 안내.
6. **코드 컨벤션 & 협업 규칙**: 우리가 중요하게 생각하는 코드 스타일과 Git 전략.
7. **핵심 비즈니스 로직**: 이 프로젝트에서 가장 중요하게 봐야 할 코드 위치나 흐름 설명.
8. **도움이 필요할 때**: 문제가 생기면 어디에 질문하거나 확인해야 하는지 안내.

# Context
- 현재 폴더 구조와 주요 파일들을 읽고 실제 구현된 내용을 바탕으로 작성해줘.
- 추상적인 설명보다는 실제 파일명이나 함수명을 인용해서 구체적으로 작성해줘.

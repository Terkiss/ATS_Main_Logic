## 🔭 2. 프로젝트 한눈에 보기
**TeruTeru Server**는 단순한 서버가 아니에요. 고성능 비동기 통신(**IOCP**) 기술을 기반으로 하는 **C# 서버 엔진**이자, 딥러닝(YOLO 등) 비전 분석을 완벽하게 지원하는 **AI 호스팅 플랫폼**입니다. 🤖

우리는 "어떻게 하면 수많은 연결을 끊김 없이 처리하면서도, 개발자가 비즈니스 로직에만 집중할 수 있을까?"라는 고민에서 시작했어요. 그 결과로 탄생한 것이 바로 **Architecture 2.0**입니다. 특히 서버를 끄지 않고도 코드를 바꿀 수 있는 **핫로딩(Hot-Reloading)** 기능은 우리 프로젝트의 자랑이랍니다! 🌟

## 🛠️ 3. 기술 스택 (우리가 이 친구들을 선택한 이유!)
우리는 단순히 유행하는 기술이 아니라, 서비스의 본질을 해결해 줄 수 있는 도구들을 골랐어요.

- **C# / .NET 9.0** 💎: 강력한 타입 안전성과 세계 최고 수준의 비동기 프로그래밍(`async/await`) 지원 덕분에, 복잡한 서버 로직을 빠르고 안전하게 짤 수 있어요.
- **IOCP (SocketAsyncEventArgs)** ⚡: 닷넷의 고성능 네트워크 IO 기술을 직접 활용해요. 덕분에 적은 자원으로도 수만 명의 손님을 동시에 맞이할 수 있죠.
- **AI-Ready (TorchSharp, OpenCV)** 🧠: 서버 엔진 안에서 직접 파이토치 모델을 돌리고 영상을 처리해요. Python으로 데이터 통신하는 오버헤드를 줄이기 위한 선택이었답니다.
- **JWT & P2P** 🛡️: 보안을 위해 **JSON Web Token**을 사용하고,Peer 간 직접 통신(**Hole Punching**)을 통해 서버 부하를 줄이고 반응 속도를 극대화했어요.

## 📁 4. 폴더 구조 파헤치기 (우리의 4계층 지도)
우리 프로젝트는 역할에 따라 아주 정교하게 나누어져 있어요. 마치 잘 정리된 서랍장 같죠! 🗄️

1. **`TeruTeruServer.SDK`** (공용 도구함) 📦
   - 프로젝트의 '약속'들이 담긴 곳이에요. 모든 프로젝트가 이 SDK를 참고하죠.
   - `ILogicService` 인터페이스나 패킷을 주고받을 때 쓰는 `CommonProtocols` 등이 여기 있어요.

2. **`TeruTeruServer.Runtime`** (엔진/심장) ⚙️
   - 서버가 실제로 살아서 움직이게 만드는 핵심 부품들이에요.
   - `MainServer.cs`가 손님을 맞이하고, `PacketPipeline`이 들어온 가방(패킷)을 검사한답니다.

3. **`TeruTeruServer.Logic.Default`** (사용자 구현 공간) 🎨 **(🔥 여기가 여러분의 주무대!)**
   - 유저 로그인, 게임 규칙, AI 분석 요청 처리 등 실제 서비스 로직이 들어가는 곳이에요.
   - **핫로딩**의 주인공이기도 해서, 여기서 코드를 고치고 빌드하면 서버 재시작 없이 바로 반영돼요!

4. **`TeruTeruServer.Cli`** (얼굴/진입점) 🖥️
   - 서버를 실행하는 '런처' 역할을 해요. `Program.cs`가 여기서 서버 엔진을 깨우고 로그를 보여준답니다.

5. **`TeruTeruServer.Client`** (손님용 가방) 🎒
   - 우리 서버에 접속하려는 앱들이 쓰는 도구예요. `TeruClient.cs`를 쓰면 복잡한 통신 로직 없이도 쉽게 서버와 연결할 수 있어요.

## 🚦 5. 시작하기 (Getting Started)
자, 이제 서버를 직접 띄워볼까요? 생각보다 아주 간단해요! 🚀

1. **전체 빌드하기**
   ```bash
   dotnet build
   ```
2. **서버 실행하기**
   ```bash
   dotnet run --project TeruTeruServer.Cli
   ```
3. **핫로딩 체험하기! ✨**
   - 서버를 켠 상태에서 `TeruTeruServer.Logic.Default/LogicPlugin.cs`의 인사말을 살짝 바꿔보세요.
   - 그리고 그 프로젝트만 빌드(`dotnet build TeruTeruServer.Logic.Default`)하면? 
   - 짠! 서버 로그에 "Hot-reloaded successfully"가 뜨면서 내가 고친 코드가 바로 적용될 거예요!

## 🤝 6. 코드 컨벤션 & 협업 규칙 (아름다운 코드 만들기)
우리는 서로의 코드를 사랑하고 아껴야 해요. 그래서 몇 가지 약속을 했답니다. 🤙

- **Naming**: 클래스와 메서드는 `PascalCase`, 변수는 `camelCase`, private 필드는 `_underScoreCamelCase`를 사용해요.
- **Why**: "무엇"을 하는 코드인지보다 "왜" 이렇게 짰는지 주석으로 남겨주면 동료들에게 큰 힘이 된답니다.
- **Git**: 새로운 기능을 만들 때는 꼭 브랜치를 새로 따주시고, 커밋 메시지는 정중하고 명확하게 작성해 주세요. 😊

## 🧠 7. 핵심 비즈니스 로직 흐름 (데이터의 여행)
데이터가 서버에 들어오면 어떤 과정을 거치는지 알면 코드가 더 잘 보일 거예요! 🗺️

1. **`MainServer.cs`**: 클라이언트의 접속을 승인해요.
2. **`PacketPipeline`**: `Validation -> Decryption -> Auth -> Routing` 순서로 보안 검색을 해요.
3. **`LogicPlugin.cs`**: 보안 검색을 통과한 패킷이 마지막으로 도착하는 종착점이에요. 
   - `ProcessJsonProtocol`: 회원가입/로그인 같은 일반적인 요청을 처리해요.
   - `ProcessDirectProtocol`: 이미지나 대용량 바이너리 데이터를 처리해요.
4. **`PluginManager`**: 뒤에서 묵묵히 지켜보다가 새로운 DLL이 감지되면 로직을 조용히 바꿔치기(Hot-Reload)해 줍니다. 🕵️
